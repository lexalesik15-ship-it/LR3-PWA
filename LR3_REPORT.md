## Назначение основных свойств manifest.json

- `name` - полное название приложения.
- `short_name` - короткое имя приложения, которое отображается на рабочем столе или домашнем экране.
- `icons` - набор иконок разных размеров для разных устройств.
- `start_url` - страница, которая открывается при запуске установленного PWA.
- `scope` - область URL, в рамках которой приложение считается PWA.
- `display` - режим отображения приложения; `standalone` открывает его как отдельное приложение без интерфейса браузера.
- `orientation` - предпочтительная ориентация экрана.
- `background_color` - цвет фона при запуске.
- `theme_color` - цвет темы браузера/системного интерфейса.
- `description` - описание приложения.
- `dir` - направление текста.
- `lang` - язык приложения.

## Характеристики события fetch

Событие `fetch` возникает в Service Worker при каждом сетевом запросе страницы. Оно позволяет перехватывать запросы, отдавать данные из кэша, выполнять запрос в сеть или комбинировать эти подходы. В работе используется стратегия: сначала поиск ответа в кэше, затем запрос в сеть.

## Запуск

```bash
npm i
npm start
```

После запуска откройте приложение в браузере, проверьте вкладку Application в DevTools: Manifest и Service Workers.
